About This Course

Welcome to the Professional Introduction to React.js course! This program is your gateway to building powerful and efficient single-page applications. If you're comfortable with HTML, CSS, and intermediate JavaScript (especially DOM manipulation and ES6 features), and you're ready to step into the world of modern front-end frameworks, this course is designed for you.

React.js, developed by Facebook, is a declarative, component-based JavaScript library for building user interfaces. This course will guide you through its core concepts, from setting up your development environment to building complex UIs with reusable components, managing application state, and interacting with data. We'll emphasize practical application through hands-on coding, exercises, and a final project to ensure you gain real-world experience.

By the end of this course, you'll have a solid foundation in React, enabling you to build scalable and maintainable web applications, and preparing you for more advanced React topics or other front-end frameworks.

Learn by Example: Practical React.js Concepts

To truly grasp React, you need to write code. These examples will walk you through fundamental React concepts, showing you how to build reusable UI components and manage their behavior.

Example 1: Basic Functional Component & JSX

This is the simplest form of a React component. It's a JavaScript function that returns JSX (which looks like HTML) to describe what the UI should look like.


// App.js (main component file)
import React from 'react'; // Always import React when using JSX

function WelcomeMessage() {
  const userName = "Learner"; // A simple JavaScript variable

  // JSX: Looks like HTML but is JavaScript
  return (
    <div className="welcome-card">
      <h2>Hello, {userName}!</h2> {/* Use curly braces for JS expressions */}
      <p>Welcome to your first React component.</p>
    </div>
  );
}

export default WelcomeMessage; // Export the component to be used elsewhere

/* Basic CSS (index.css or App.css) */
.welcome-card {
  background-color: rgba(255, 255, 255, 0.15);
  padding: 30px;
  border-radius: 10px;
  text-align: center;
  margin: 20px auto;
  max-width: 400px;
  box-shadow: 0 4px 15px rgba(0, 0, 0, 0.2);
}
.welcome-card h2 {
  color: var(--accent-color);
  margin-bottom: 10px;
}
.welcome-card p {
  color: var(--text-color);
}

What's happening here? We define a JavaScript function `WelcomeMessage`. Inside, we return JSX, which looks like HTML but allows us to embed JavaScript expressions (like `{userName}`) using curly braces. `export default` makes this component available for other files to import and use.

Example 2: Passing Data with Props

Props (short for properties) are how you pass data from a parent component to a child component. They are read-only, ensuring a predictable data flow.


// UserCard.js (Child Component)
import React from 'react';

function UserCard(props) { // props is an object containing all passed properties
  return (
    <div className="user-card">
      <h3>Name: {props.name}</h3> {/* Access properties via props object */}
      <p>Age: {props.age}</p>
      <p>Occupation: {props.occupation}</p>
    </div>
  );
}

export default UserCard;

// App.js (Parent Component)
import React from 'react';
import UserCard from './UserCard'; // Import the child component

function App() {
  return (
    <div className="app-container">
      <h1>User Profiles</h1>
      <UserCard name="Alice" age={30} occupation="Software Engineer" /> {/* Pass data as attributes */}
      <UserCard name="Bob" age={24} occupation="Graphic Designer" />
      <UserCard name="Charlie" age={35} occupation="Project Manager" />
    </div>
  );
}

export default App;

/* Basic CSS (App.css or index.css) */
.app-container {
  text-align: center;
  margin-top: 20px;
}
.user-card {
  background-color: rgba(255, 255, 255, 0.1);
  padding: 20px;
  border-radius: 10px;
  margin: 15px auto;
  max-width: 300px;
  box-shadow: 0 3px 10px rgba(0, 0, 0, 0.1);
  text-align: left;
}
.user-card h3 {
  color: var(--accent-color);
  margin-bottom: 5px;
}
.user-card p {
  margin-bottom: 5px;
}

What's happening here? The `App` component is the parent. It renders multiple `UserCard` components. When rendering `UserCard`, we pass data like `name`, `age`, and `occupation` as attributes. Inside `UserCard`, these attributes are received as a single `props` object, and we can access them using `props.name`, `props.age`, etc. This demonstrates how data flows down the component tree.

Example 3: Managing State with the `useState` Hook

The `useState` hook allows functional components to have "state" â€“ data that changes over time and causes the component to re-render. It's fundamental for interactive UIs.


// Counter.js (Component with State)
import React, { useState } from 'react'; // Import useState hook

function Counter() {
  // Declare a state variable 'count' and a function 'setCount' to update it
  // Initial value of count is 0
  const [count, setCount] = useState(0);

  const increment = () => {
    setCount(count + 1); // Update state, React re-renders component
  };

  const decrement = () => {
    setCount(count - 1); // Update state
  };

  return (
    <div className="counter-app">
      <h2>Count: {count}</h2>
      <button onClick={increment}>Increment</button>
      <button onClick={decrement}>Decrement</button>
    </div>
  );
}

export default Counter;

/* Basic CSS (App.css or index.css) */
.counter-app {
  background-color: rgba(255, 255, 255, 0.15);
  padding: 30px;
  border-radius: 10px;
  text-align: center;
  margin: 20px auto;
  max-width: 400px;
  box-shadow: 0 4px 15px rgba(0, 0, 0, 0.2);
}
.counter-app h2 {
  color: var(--accent-color);
  margin-bottom: 20px;
}
.counter-app button {
  padding: 10px 20px;
  margin: 0 10px;
  background-color: var(--accent-color);
  color: #333;
  border: none;
  border-radius: 5px;
  cursor: pointer;
  transition: background-color 0.3s ease;
}
.counter-app button:hover {
  background-color: #ffc107;
}

What's happening here? `useState(0)` initializes `count` to `0` and gives us `setCount` to update it. When `setCount` is called, React automatically re-renders the `Counter` component, displaying the new `count` value. This is how React handles dynamic data and updates the UI efficiently.

Example 4: Conditional Rendering

React allows you to render different elements or components based on certain conditions. This is crucial for showing/hiding UI elements, handling loading states, or displaying different content based on user roles.


// Greeting.js (Component with Conditional Rendering)
import React, { useState } from 'react';

function Greeting() {
  const [isLoggedIn, setIsLoggedIn] = useState(false); // State to track login status

  const toggleLogin = () => {
    setIsLoggedIn(!isLoggedIn); // Toggle the boolean state
  };

  return (
    <div className="greeting-card">
      {isLoggedIn ? ( // Conditional rendering using a ternary operator
        <h2>Welcome Back!</h2>
      ) : (
        <h2>Please Log In</h2>
      )}
      <button onClick={toggleLogin}>
        {isLoggedIn ? 'Log Out' : 'Log In'}
      </button>
      {isLoggedIn && <p>You are currently logged in.</p>} {/* Conditional rendering with && operator */}
    </div>
  );
}

export default Greeting;

/* Basic CSS (App.css or index.css) */
.greeting-card {
  background-color: rgba(255, 255, 255, 0.15);
  padding: 30px;
  border-radius: 10px;
  text-align: center;
  margin: 20px auto;
  max-width: 400px;
  box-shadow: 0 4px 15px rgba(0, 0, 0, 0.2);
}
.greeting-card h2 {
  color: var(--accent-color);
  margin-bottom: 15px;
}
.greeting-card button {
  padding: 10px 20px;
  background-color: var(--accent-color);
  color: #333;
  border: none;
  border-radius: 5px;
  cursor: pointer;
  transition: background-color 0.3s ease;
}
.greeting-card button:hover {
  background-color: #ffc107;
}
.greeting-card p {
  margin-top: 15px;
  font-style: italic;
}

What's happening here? We use the `isLoggedIn` state to control what is rendered. The ternary operator (`condition ? true_render : false_render`) is used to show different `h2` texts. The logical `&&` operator (`condition && render_if_true`) is used to conditionally render the paragraph only when `isLoggedIn` is true. This pattern is essential for creating dynamic UIs that adapt to application state.
